How javaScript Works?

javaScript is a single threded interpreted programming language. There is a javaScript engine which executes the code. When browser gets a JS code, Javascript engine (V8 for Chrome) parse the code and makes a syntax tree. After making syntax tree, JS engine start the interpreter which is called interpreter ignition. Javascript engine also uses a Just in Time (JIT) compiler to compile the code in real time. It also uses a optimizer to optimize the machine code code which is known as TurboFan. Then it converts the machine code into bytecode. The javascript engine consist of two components: 1) Memory Heap: Here all the memory allocation operations happens, 2) Call Stack: Here all the function calling mechanism happens. The call stack is where the code actually executed. The heap stores all the objects in our code.

What does JS event loop do?

Event loop in javaScript is a process that handles the communication between the Call Stack (synchronous events) and Callback Queue (asynchronous events). It constantly checks if the call stack is empty or not. If empty then new functions are added from the callback queue. The javascript engine (V8 for chrome) consist of two parts, Heap and Call Stack. Browsers assist the engine with Web API's and Callback Queue. When JS engine works with asynchronous code the web api handles the blocking and non blocking behaviour of javaScript code. The Callback Queue built with all the synchronous callback functions and they wait here mto go back to the call stack for execution.The callback Queue works using first in first out (FIFO) method.
